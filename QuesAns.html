Question - 2 : What is the difference between innerHTML, innerText, and textContent ?

Difference between innerText, innerHTML and textContent is given below,
innerText :
It only retrieves the text content of an element. Whitespace is trimmed off automatically. If an element is (display = 'none') then it ignores that content.

innerHTML :
It retrieves the "HTML" content and also allows us to insert an "HTML" content dynamically and also can't trim Whitespace. Very useful to manipulate any "HTML" content.

textContent :
It is used to retrieve only text contents. It can't trim Whitespace. Even "HTML" contents are also treated as plain text.

Question - 3 : What is event delegation in the DOM?

It is a javascript technique to reduce frequent occurance of the same task. In this technique instead of adding event listeners to multiple child elements, we attach a single event listener to a parent element and handle events for child elements using event bubbling.

Question - 4 : What is event bubbling in the DOM?

Event bubbling is a mechanism in the DOM (Document Object Model) where an event starts from the target element and propagates up through its parent elements until it reaches the root or to the "HTML".

Question - 5 : How do you create, add, and remove elements using JavaScript?

In JavaScript, dynamically manipulating the DOM allows you to create, add, and remove elements from a webpage. To create an element, you can use document.createElement(tagName), which generates a new element of the specified type. Once created, the element can be inserted into the DOM using methods like appendChild(), which adds it at the end of a parent, prepend(), which places it at the beginning, or insertBefore(), which allows precise placement before another element. Additionally, innerHTML can be used to insert multiple elements at once but is less efficient for frequent updates. To remove elements, removeChild() is used when deleting a specific child from a parent, while remove() can directly eliminate an element. If you need to clear an entire section, setting innerHTML = "" on a parent container effectively removes all its child elements. These techniques are fundamental for dynamic UI updates, allowing developers to modify web pages without requiring a full reload.
